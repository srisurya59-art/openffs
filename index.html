Operational & Compliance Overview

Page 1 — Standards Foundation

IntegriFFS implements the unified Fitness-for-Service methodology of API 579-1/ASME FFS-1 (2021 Edition), the industry standard referenced across the integrity management lifecycle:

  • API 570 — Piping Inspection Code (invokes API 579 when FFS required)
  • API 510 — Pressure Vessel Inspection Code (invokes API 579 for fitness evaluations)
  • API 653 — Tank Inspection, Repair, Alteration, Reconstruction (invokes API 579 for shell evaluations)
  • ASME B31.3 / B31.1 — Process / Power Piping (FFS per API 579)

Page 2 — Algorithmic Verification Protocol

All calculations follow a deterministic, reproducible engineering workflow:

  1. Operator inputs material properties, geometry, and damage data
  2. Engine computes Required Thickness (t_min) per ASME Section VIII Div 1 UG-27
  3. Engine computes Maximum Allowable Working Pressure (MAWP)
  4. Engine compares measured thickness against t_min — Pass or Fail verdict
  5. PDF compliance certificate generated with operator logo and full audit trail

Note: IntegriFFS implements Part 4 (General Metal Loss) of API 579. It is independently developed and not certified by API or ASME. Users must verify results against their own engineering judgment.
